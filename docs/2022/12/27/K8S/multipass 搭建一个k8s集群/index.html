四、创建一个pod

一次deployment为一次部署,当外部client访问服务器时，首先访问服务器一致对外提供的接口，通过这一接口再将请求workload到每台主机上，每台主机上都有相对应的NodePort端口来接受这一请求，每个service服务上都有一个ClusterIP,通过ClusterIP和每台主机上的IPTables表，service将求情workload到具体的每个pod上。
所以需要创建两个yaml, nginx.yaml用于创建单个或者多个pod， nginx.service.yaml用于暴露pod给外界调用

  • 创建一个nginx.yaml

  • 创建一个nginx.service.yaml

  • 创建pod

  • 查看pod

  • 查看service

  • 访问nginx

    通过节点任意节点地址+service映射的端口”8080:30062/TCP”即可访问nginx

五、填坑记录

查看错误信息

  • 1： Container runtime network not ready: NetworkReady=false reason:NetworkPluginNotReady message:docker: network plugin is not ready: cni config uninitialized Unable to update cni config: No networks found in /etc/cni/net.d Failed to get system container stats for “/system.slice/kubelet.service”: failed to get cgroup stats for “/system.slice/kubelet.service”: failed to get container info for “/system.slice/kubelet.service”: unknown container “/system.slice/kubelet.service”

    解决方案：安装kubernetes-cni 在重启

  • 2：NetworkPlugin cni failed to set up pod “mysql-mb6t7_default” network: open /run/flannel/subnet.env: no such file or directory 1608 cni.go:228] Error while adding to cni network: open /run/flannel/subnet.env: no such file or directory

    解决方案：主节点及子节点增加文件 /run/flannel/subnet.env

  • 3: The connection to the server localhost:8080 was refused - did you specify the right host or port?

    解决方案： 出现这个问题的原因是kubectl命令需要使用kubernetes-admin的身份来运行，在“kubeadm int”启动集群的步骤中就生成了”/etc/kubernetes/admin.conf”
    修改admin.conf并复制到子节点下

    最后再添加环境变量即可

一、Monorepo介绍

Monorepo是一种项目管理方式，通常是将多个项目的代码放在一个仓库中，这就导致重复的代码及工具会有很多，当前便有一些Monorepo的工具可使用，一定程度解决这些问题。

  • 比如以下几种工具
工具 介绍
Bit 用于驱动开发的工具链
Turborepo 用于JS和TS的高性能构建系统
Rush 一个可扩展的web单仓库处理器
Nx 具有一流的 monorepo 支持和强大集成的下一代构建系统
Lerna 用来管理多个软件包项目

二、TurboRepo介绍

是一个构建JS、TS的Monorepo高性能构建系统，抽象出复杂的配置、脚本、工具，减少了项目的复杂性，使得专注于业务代码的开发。
TurboRepo支持npm/pnpm/yarn，建议使用pnpm。

三、TurboRepo优点

  • 支持多任务运行

    TurboRepo支持多任务运行，可同事进行多项目的运行和编译，下图可见其运行优势

  • 增量构建

    如果我们的项目过大，构建多个子包会造成时间和性能的浪费，turborepo中的缓存机制 可以帮助我们记住构建内容 并且跳过已经计算过的内容，优化打包效率。

  • 简单配置

    通过约定的配置方式，简单的配置降低复杂性，只需几行JSON 即可配置整个项目依赖，执行脚本的顺序结构。

四、TurboRepo核心概念

  • 管道

    TurboRepo为开发人员提供了一种以常规方式显式指定任务关系的方法，turbo在定义monorepo的任务依赖关系图，我们需要在根目录定义turbo.json 执行各种调度，输出，缓存依赖， 打包等功能。

turbo.json位于TurboRepo项目根目录, 如下图所示在pipeline中的每一个key都指向我们在package.json中定义的script脚本执行命令,并且在pipeline中的每一个key
都是可以被turbo run所执行 执行pipeline的脚本的名称。您可以使用其下方的键以及与缓存相关的一些其他选项来指定其依赖项，

在我们执行turbo run ***命令的时候 turbo 会根据我们定义的 Pipelines 里对命令的各种配置去对我们的每个package中的package.json 中 对应的script执行脚本进行有序的执行和缓存输出的文件。

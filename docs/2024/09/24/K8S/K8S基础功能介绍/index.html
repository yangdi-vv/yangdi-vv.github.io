K8S介绍

Kubernetes是一个强大的容器编排系统，它允许用户更加方便地部署、扩展和管理容器化应用程序。它提供了一系列资源管理机制，如资源调度、容器网络和存储编排等，确保容器集群的高可用性和可靠性。

功能使用

1、Ingress

  • 介绍

    在Kubernetes集群中，Ingress作为集群内服务对外暴露的访问接入点，几乎承载着集群内服务访问的所有流量。Ingress是Kubernetes中的一个资源对象，用来管理集群外部访问集群内部服务的方式。可以通过Ingress资源来配置不同的转发规则，从而实现根据不同的规则设置访问集群内不同的Service所对应的后端Pod。

2.service

  • 介绍

    Service在Kubernetes中是一个核心概念，它从逻辑上代表了一组Pod，并为这组Pod提供了一个单一的、固定的网络端点，如IP地址和端口。这样，无论后端Pod如何变化（例如增减或故障替换），客户端都可以通过同一个Service来访问服务，而无需关心具体的Pod实例。

3. deployment

  • 介绍

    Deployment是一个Kubernetes资源，用于声明期望的Pod状态，并确保实际状态与此相符。它通过创建和管理ReplicaSet来实现这一点，从而保证始终有正确数量的Pod副本在运行。

4.configmap

  • 介绍

    ConfigMap是一个API对象，用于以键值对的形式存储非敏感的配置信息，如环境变量、配置文件等。它允许用户将配置信息从容器镜像中解耦出来，使得配置的更新和管理更加灵活和便捷。

5.nameSpace

  • 介绍

    Namespace（命名空间）是对一组资源和对象的抽象集合，比如可以将系统内部的对象划分为不同的项目组或用户组。在Kubernetes集群中，可以存在多个逻辑上彼此隔离的命名空间，从而实现资源的隔离和管理。

6.secret

  • 介绍

    Secret是用来保存和传递密码、密钥、认证凭证等敏感信息的Kubernetes资源对象。它允许用户将敏感数据从容器镜像或Pod规范中解耦出来，使得配置的更新和管理更加灵活和安全。
